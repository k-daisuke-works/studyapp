# カードフォーマット定義

用語カードの各フィールドを書くときはこの定義に従うこと。

---

## d（技術的定義）

試験で問われる正確な定義を1〜2文で書く。

- 規格名・正式名称から始める（例：「IEEE 754 で定めた 32 ビット浮動小数点数。」）
- 構成要素は「A・B・C」形式で列挙する
- ビット数・桁数・範囲など試験に出る数値を必ず含める
- 体言止めで締める

**例：**
> IEEE 754 で定めた 32 ビット浮動小数点数。符号 1 ビット・指数 8 ビット・仮数 23 ビットで構成され、有効桁約 7 桁、表現範囲は約 10⁻³⁸〜10³⁸。

---

## easy（やさしく言うと）

情報I履修済みの高校生向けに100〜200字でイメージさせる。

- 「コンピュータが〜するとき、〜だから」と背景・制約から始める
- 「〜という方式のこと」「〜できる」で締める
- 比喩・身近な例で「何が嬉しいか」を伝える
- 「試験では」「選択肢を判別」は禁止

**例：**
> コンピュータが数を保存するとき、メモリの容量が限られているから、小数の桁数を制限しないといけない。単精度は32ビットのメモリを使って、約7桁の精度で数を表す方式のこと。

---

## example（現実の使用場面）

**重要：抽象的な数値例だけで説明しないこと。「現実のシステムや業務で、なぜその処理が必要になるのか」から説明すること。**

**題材は必ずゲーム・ゲーム業界から選ぶこと。**
- 基礎理論・計算系 → ゲームエンジンの内部処理、スコア・グラフィック計算
- ネットワーク・セキュリティ → FF14・原神などのオンラインゲームの通信・認証
- データベース → プレイヤーデータ・アイテム・ランキング管理
- プロジェクト管理 → ゲーム開発プロジェクト（スクエニ・任天堂規模）
- 経営・法務 → ゲーム会社の経営・ガチャ規制・著作権
- ゲームで直接例えにくい場合も、ゲーム業界・ゲーム会社の業務を軸にする

各セクションは**1点につき1行・体言止め**でまとめる。説明が長くなりそうなら削る。「冗長・複雑・難しい」と感じたら書きすぎのサイン。

以下の順番・形式で書く：

### ■ 現実の使用場面
**箇条書き**で書く。大きな目的から小さな目的へ順に絞り込む4点構成。

```
- 大目標：〜を実現する（最終的に何をしたいか）
- 小目標：そのために〜を行う（具体的な処理の目的）
- 活用：この用語を使って〜する
- なぜ：〜だからメモリ節約・高速化・安全性向上などができるため
```

### ■ 処理前の状態
**`|` 区切りのテーブル**でラベル付きデータを示す。テーブル後の `→` は次のセクションへの橋渡しとして使う。

- ビット位置を示す場合は「ビット0（1桁目）」のように括弧で桁数を補足する

```
| データ | 値 | 補足 |
|---|---|---|
| ラベル付きデータ | 具体的な値 | 単位・規模 |

→ この状態を〜で表すと〜（次セクションの出発点になる値を示す）
```

### ■ 仕組みの中身
導入文で「大きな目的 → この用語を使う理由」を先に書いてからテーブルに入る。目的なしにいきなりステップを始めない。

テーブルの後の `→` で計算の答えと意味を明示する。複数の補足は `→` を1行ずつ分けて書く。

```
〜するために〜が必要で、この用語を使うと〜できる。

| ステップ | 操作 | 値 | 何が起きているか |
|---|---|---|---|
| 具体値 | ... | ... | ... |

→ 計算の答え = X → Xの意味（Y なら〜）
→ 使った演算の性質や注意点
→ 関連用語との違い（[[関連用語]]）
```

### ■ 処理後の結果
**箇条書き**で3点を書く。

```
- 期待：〜できる
- 実際：〜のため〜になる
- 問題：〜が致命的になるため〜が必要
```

**クロスリンク：**
- 本文・表のセル内を問わず、登場する用語は `[[用語名]]` でリンクする
- カードあり→クリッカブル、なし→※注釈として自動表示される

**禁止：**
- 目的を省略してステップだけ説明しない（なぜその処理をするのかを必ず先に書く）
- A=1.234567 のような意味のない数値だけで説明しない
- 定義だけで終わらせない
- 途中過程を省略しない
- 「だから何に使うのか」が分からない説明にしない

## ex（図解・例示）

detail カード専用。用語の実体を図や表で可視化する。

- 最初の1文：具体的なシナリオを書く（「スマホの3Dゲームで〜するとき」「銀行で〜を計算するとき」）
- 次の行：`【これが[[用語名]]（補足）】` ヘッダー
- ｜区切りテーブルで構造・対比・ビット列などを表現する（各列にラベルを付ける）
- 空行を挟んで技術的な補足説明を加える
- `[[用語名]]` でクロスリンクを積極的に挿入する

**例：**
```
スマホの3Dゲームでキャラクターの座標を「X=1234.567」として保存するとき、コンピュータは32個の0と1だけで表現する。

【これが[[単精度浮動小数点数]]（32ビット）】
  符号(1ビット) ｜ 指数部(8ビット) ｜ 仮数部(23ビット)
     +          ｜   10001001    ｜  00110100010...
  （プラス）     ｜ （スケール）   ｜ （数字の中身・約7桁）

たった32ビットで約10⁻³⁸〜10³⁸の範囲を表せる。ただし[[仮数]]部が23ビットしかないため有効桁は約7桁。
```
